Die wichtigsten Aufgaben werden im Kapitel Browser Modus Menübefehle beschrieben. Im Folgenden möchte ich auf spezifische Bereiche von PhoA eingehen: {[%hh_h2:Bilddaten]}

Wie bereits im Übersichtsschema dargestellt wurde, speichert das Programm Bilderlisten einzeln ab (genau gesagt handelt es sich um Listen, der die Elemente der Bilddateien inkl. Beschreibung, Miniaturansichten etc. zugeordnet sind). Die Baumstruktur wird ebenso seperat gespeichert. Jede einzelne Gruppe beeinhaltet eine Reihe von Links zu den Bildern, wenn Sie also eine Datei einer Gruppe zuordnen (da ja alle Bilder Dateien darstellen) und diese Datei bereits irgendwo im Fotoalbum angelegt und beschrieben ist, findet das Programm automatisch die dazugehörigen Daten. Dasselbe gilt für den Fall, dass Sie Bilddaten bearbeiten (etwa den Ort oder das Datum ändern) - die Daten werden in allen dazugehörigen Bereichen automatisch geändert!

Anders verhält es sich, wenn Sie ein Bild aus einer Gruppe entfernen wollen. Ist das zu löschende Bild noch anderen Gruppen im Fotoalbum zugeordnet, wird nur der Link zum Bild entfernt, das Bild selbst ist im Fotoalbum immer noch vorhanden (nur eben anderen Gruppen zugeordnet). Wollen Sie diese Bilddatei später erneut einer Gruppe hinzufügen, wird PhoA alle verfügbare Daten dieses Bildes der Datei automatisch zuordnen. Löschen Sie jedoch ein Bild, das nur der aktuellen Gruppe zugeordnet ist, wird die Bilddatei vollständig gelöscht. Fügen Sie das Bild später wieder ein, müssen Sie die entsprechenden Daten selbst eingeben. {[%hh_h2:Drag & Drop]}

Natürlich kann ein Bild in einer Gruppe nicht zwei Mal vorhanden sein. Am einfachsten können Bilder in eine andere Gruppe kopiert werden, indem man sie mit der Maus vom rechten Fenster (dem Thumbnail-Bereich) in die entsprechende Gruppe der Baumansicht zieht und dabei die Strg-Taste gedrückt hält. Ohne Drücken der Strg-Taste wird das Bild nicht kopiert, sondern nur verschoben (von der Ausgangsgruppe in die Zielgruppe). Es können mehrere Bilder gleichzeitig kopiert bzw. verschoben werden: Als erstes müssen Sie die zu kopierenden Bilder markieren (verwenden Sie dazu die Tastenkombination Strg, Shift und die linke Maustaste). Nehmen Sie dann ein markiertes Bild (egal welches) und ziehen Sie es hinüber. Nachdem die Dateien verschoben oder kopiert wurden, zeigt PhoA in einem Dialogfenster die durchgeführten Änderungen an. (Dazu muss diese Funktion im Menü Tools | Optionen Feld (Dialoge | Benachrichtigungen)aktiviert sein).

Die Drag & Drop-Funktion ist auch auf ganze Ordner anwendbar, die Baumansicht wird dadurch neu geordnet. Wenn Sie eine Gruppe in den freien Bereich rechts von den Verknüpfungen der Baumstruktur (Knotenpunkte) ziehen, wird der Ort, an dem die Gruppe eingefügt wird Einfügeort als Linie angezeigt. Die Gruppe wird an gleichrangiger Stelle wie der Zielknotenpunkt eingefügt, über oder unter den Punkt, je nach dem, wohin Sie Maus bewegen (dies beeinflusst ebenfalls das Aussehen der Einfügelinie). Ziehen Sie eine Gruppe auf ein andere, wird diese markiert und die Gruppe wird zu deren Untergruppe. Beachten Sie: Die Baumstruktur kann nur Fotoalbum-Ordner enthalten, andere Ordner können nicht eingefügt werden!

In ähnlicher Weise können Sie Bilder kopieren (nicht jedoch verschieben), die Sie durch die Suchfunktion erhalten haben (Menü Tools | Suchen). Es ist empfehlenswert, Bilder nach Datum (z.B. nach Jahr oder nach Monat) zu sortieren, nachdem diese zuvor in einem anderen Ordner nach Thema sortiert haben (Benutzen Sie dazu die Suchfunktion um Bilder nach Datum zu suchen und kopieren Sie dann die Ergebnisse in die entsprechende Gruppe).

Mit Drag & Drop können Sie aber auch Bilder innerhalb einer Gruppe neu ordnen: Ziehen Sie einfach ein markiertes Thumbnail an eine andere Stelle im Thumbnail-Fenster, dabei wird die Einfügelinie angezeigt. Wenn Sie die Maustaste loslassen, werden alle markierten Bilder an den angegebenen Ort verschoben; die Reihenfolge der Bilder wird dabei nicht verändert. Eine weitere Methode, Bilder neu zu ordnen, besteht darin, diese zu sortieren. {[%hh_h2:Zwischenablage]}

PhoA unterstützt die Zwischenablage-Funktion. Zur Zeit können nur Bilder von Phoa ausgeschnitten, kopiert und eingefügt werden. Daten, die in die Zwischenablage eingefügt werden, bleiben vollständig erhalten - mit Ausnahme der Angabe, welcher Gruppe die Bilder zugeordnet wurden. Durch diese Funktion können Bilder in eine andere Gruppe oder auch in ein anderes Fotoalbum verschoben oder kopiert werden.

Was das Einfügen von Bildern in ein anderes Fotoalbum betrifft, gibt es jedoch gewisse Einschränkungen: Daten von bereits vorhandenen Bildern (Bilder, die unter dem selben Dateinamen bereits im Fotoalbum gespeichert sind) werden nicht überschrieben. Zum Beispiel: Sie fügen ein Bild, das unter C:\Fotos\Winter-Urlaub-1812.jpg gespeichert ist, in das Fotoalbum ein. Besteht dieses Bild im Fotoalbum bereits, wird das Bild nur der aktuellen Gruppe hinzugefügt (falls es dort nicht schon ist). Ist das neu eingefügte Bild im Zielordner bereits vorhanden, wird dieses Bild einfach übersprungen. Nach Beendigung dieses Vorgangs benachrichtigt Sie Phoa über die Anzahl an Bildern, die soeben aus der Zwischenablage eingefügt wurden. (Dazu muss diese Funktion im Menü Tools Optionen Feld Dialoge Benachrichtigungen aktiviert sein. {[%hh_h2:Rückgängig-Funktion]}

Im Gegensatz zu vielen ähnlichen Produkten, unterstützt PhoA vollständig die Rückgängig-Funktion. Mit Hilfe dieser Funktion kann ein im Fotoalbum vorgenommener Arbeitsschritt rückgängig gemacht werden (einzige Ausnahme bildet der Befehl Thumbnails erneuern) - oder aber sogar mehrere Arbeitsschritte gleichzeitig! Gehen Sie dazu auf Bearbeiten | Rückgängig bzw. Bearbeiten | Rückgängig Verlauf im Menü Bearbeiten. Wenn Sie einen Punkt aus der Liste Rückgängig-Verlauf markieren, werden alle vorhergehenden Arbeitsschritte (d.h. die, die in der Liste weiter oben sind) ebenfalls automatisch rückgängig gemacht.

Die Anzahl der Schritte, die rückgängig gemacht werden können, wird im Menü Tools | Optionen, im Dialog Browser Modus angegeben. Die maximale Anzahl der Schritte, die rückgängig gemacht werden können, beträgt 32. {[%hh_h2:Ansichten]}
