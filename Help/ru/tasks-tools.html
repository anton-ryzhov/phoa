{[%hh_hdr_ru:Типичные задачи :: Подключение внешних инструментов]}

В PhoA есть возможность подключения внешних программ для обработки, просмотра или печати изображений фотоальбома.

Это делается с помощью так называемых инструментов, которые создаются и настраиваются вами.

Существует несколько видов инструментов.

Основной вид инструментов представляют собой так называемые действия по умолчанию. Кроме того, есть возможность подключения любых программ в качестве пользовательских инструментов. {[%hh_h2:Действия по умолчанию]}

Это, если можно так выразиться, набор «упрощённых» вариантов инструментов.

В Windows, как известно, существует понятие типа файла, которое однозначно связано с расширением файла. Например, файлы с расширением .doc представляют собой тип «Документ Microsoft Word», а файлы .html — это документы Интернета.

У большинства зарегистрированных типов файлов (во всяком случае, документов) в Windows есть связанное приложение, открывающее файлы этого типа, когда вы запускаете их из Проводника. Это называется действием по умолчанию для данного типа файлов. Так, файлы .doc открываются в Microsoft Word, а для файлов .html запускается браузер по умолчанию.

Именно на такое поведение рассчитан вид инструмента Стандартное действие по умолчанию.

Кроме него, есть еще некоторый набор стандартных действий, а именно:

  • Открытие (open);
  • Редактирование (edit);
  • Печать (print).

Для каждого из этих действий в программе предусмотрен вид инструмента.

Любое из вышеперечисленных действий может быть настроено пользователем по своему вкусу с помощью стандартных средств Windows: делается это в диалоге, вызываемом выбором пункта меню Проводника Сервис | Свойства папки на вкладке Типы файлов.

Многие программы для работы с графикой (такие как графические редакторы или средства просмотра; но PhoA к ним не относится) очень любят переопределять ассоциации типов файлов «на себя». Некоторые программы предварительно об этом спрашивают.

Таким образом, если вас устраивают текущие ассоциации типов файлов, вам проще и удобнее будет воспользоваться именно такими стандартными видами инструментов.

Есть ещё одно, довольно неочевидное преимущество стандартных действий перед простым запуском программ-инструментов (описываемых ниже). Дело в том, что зачастую стандартные действия в Windows выполняются с помощью так называемых команд DDE (Dynamic Data Exchange), которые позволяют открывать несколько изображений в одном приложении, в то время как при использовании программ-инструментов зачастую для каждого изображения запускается новый, отдельный экземпляр программы, что менее удобно. {[%hh_h2:Заданное действие]}

Этот вид инструментов позволяет задать любую программу для открытия (или обработки) заданного типа файлов. Чаще всего это будет графический редактор или средство просмотра, но может быть и более специфический инструмент, например, программа пакетной обработки изображений. {[%hh_h2:Пункт-разделитель]}

Это самый незамысловатый вид инструмента и нужен он для создания разделителей в меню.

Так, с видами инструментов мы разобрались. Теперь есть смысл кратко описать {[%hh_h2:Маски инструментов]}

Чтобы задать применимость инструментов к разным видам изображений, в PhoA каждому инструменту сопоставлена так называемая маска, которая и определяет, может ли инструмент быть применён к файлам определённого типа. Правила составления масок во многом похожи на соответствующие правила Windows, и используются такие маски во многих местах программы. Подробное описание правил составления масок файлов приведено здесь.

Инструмент становится доступным только когда все выделенные изображения подходят под его маску (точнее, маски, так как их может быть несколько). {[%hh_h2:Использование инструмента]}

Чтобы инструмент был полезен, нужно, чтобы как минимум имелась возможность его вызвать. В PhoA инструменты могут появляться в различных меню. В каких именно — задаётся в его свойствах. {[%hh_h2:Настройки пользовательского действия]}

Как было сказано выше, для данного вида инструментов необходимо указать программу, а также некоторые дополнительные параметры. Подробное описание свойств инструментов приведено здесь.

Указанной программе передаётся в качестве параметра командной строки полный путь к файлу изображения. Большинство программ обработки изображений воспринимают таким образом файлы для открытия или обработки. {[%hh_seealso_ru]} Инструменты
