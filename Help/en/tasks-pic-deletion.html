{[%hh_hdr_en:Common tasks :: Picture deletion]}

It would be better to clarify some things about deleting pictures in PhoA.

As a matter of fact, photo album groups contain links to the pictures, while a picture links a corresponding disk file.

Hence when you delete a picture from a group, this does not mean the picture is removed from the album. The picture is actually removed from the photo album only when it is completely removed from all its groups.

Moreover, when you completely remove a picture from the album, this does not delete the corresponding file so it remains on the disk.

In order to find out which groups a picture is belonging to you can use the Groups page of the Picture properties dialog. And how do you remove unneeded pictures (for example, bad photos) with corresponding files is described below. {[%hh_h2:Delete picture files operation]}

To delete pictures with corresponding files PhoA provides the File operations wizard.

  1. Select thumbnails you would like to delete.
  2. Launch the File operations wizard. You do it by selecting the {[%hh_ii:FileOps]} Tools | File operations... menu item.
  3. On the first page of the wizard select Delete pictures and files and click Next.
  4. If at the moment you started the wizard the thumbnail window was active, the Selected pictures radio button will be checked by default. That is just the same you need.
  5. On the next page you can select whether files are wiped completely or just moved to the Recycle Bin (in the latter case you can undo the deletion later; but you would be able to recover deleted files, not the pictures deleted from the album). After clicking Next the program will start picture and file deletion.
