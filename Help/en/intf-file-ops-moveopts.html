{[%hh_hdr_en:Picture file operations wizard :: Copy/move options page 1]}

This wizard page configures parameters for the following file operations:

  • Copying picture files into a specified folder
  • Moving picture files into a specified folder
{[%hh_h2:Controls]}
File arranging
These options determine the way the program places files to the destination folder.
Put all the files into the destination folder This mode places all the files into a single folder - the destination folder.
NB: Please note that it is allowed for files to have the same names. This may cause a conflict when placing them into a single folder.
Maintain original folder layout In this mode the program arranges all the files duplicating their original folder structure. See below for an example.
Use path relative to PhoA determines automatically the longest common file path for all the selected pictures, and then places it and all its shortened variants into the list.
An example on placing files maintaining the original folder layout
Consider we have selected the following pictures for copy/move operation:
  • C:\My documents\Photo album\Spring 1812\pic001.jpg
  • C:\My documents\Photo album\Summer 1812\pic002.jpg
  • C:\My documents\Photo album\Autumn 1812\pic003.jpg
  • C:\My documents\Photo album\Winter 1812\pic004.jpg
  • C:\My documents\Photo album\Winter 1812\Moscow\pic005.jpg

The program automatically finds that the longest common path for all the pictures is C:\My documents\Photo album, and places it into the list of possible base folders as well as other valid choices: C:\My documents and C:\; the last choice being added is (none): when you select it the drive letter also becomes a folder inside the destination folder.

Let's assume now we have selected C:\My documents as a base folder, and D:\CD-ROM\Disk1 as a destination folder. Once the operation is complete we'll have the following set of files:

  • D:\CD-ROM\Disk1\Photo album\Spring 1812\pic001.jpg
  • D:\CD-ROM\Disk1\Photo album\Summer 1812\pic002.jpg
  • D:\CD-ROM\Disk1\Photo album\Autumn 1812\pic003.jpg
  • D:\CD-ROM\Disk1\Photo album\Winter 1812\pic004.jpg
  • D:\CD-ROM\Disk1\Photo album\Winter 1812\Moscow\pic005.jpg

The program takes D:\CD-ROM\Disk1 as a start point, and each picture file will have folder Photo album in its path as the path is relative to C:\My documents.

Arrange files into folders maintaining group layout This mode causes PhoA to arrange files into folders duplicating the existing picture group hierarchy in the photo album or view (depends on what is active at the moment).
Use path relative to group This list allows you to specify the start group from which the program copies the folder structure. The group most distant from the root (photo album or view), which is the common parent for all the selected picture groups, is selected by default (so if the photo album node itself is selected for the operation, or you have selected the whole photo album, the list will contain the only Photo album item). Also the list is populated with all the parent groups of that group including the ultimate parent of all the groups: Photo album or current view.
Allow multiple copies of the file As PhoA allows linking a picture to multiple groups, the copy operation maintaining group layout may require to place multiple copies of a file into different folders. If this check box is on the program will act just this way, otherwise the file will be copied only once (into the first folder in traversal path).
Replace unallowed symbols with the char Specifies a character to replace the unallowed in folder names characters with (these are: \ / : * ? " < > |). E.g. if this char is an underscore _ (by default), and there's a group named Friends/Enemies, then it will have the corresponding folder named Friends_Enemies.
